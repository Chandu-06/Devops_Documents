================================================================
WHAT IS VERSION CONTROL
================================================================

----------------------------------------------------------------
THE PROBLEM WITHOUT IT
----------------------------------------------------------------

Before version control existed, developers managed code by 
manually making copies of files and renaming them.

A typical project folder looked like this:

    project_final.sh
    project_final_v2.sh
    project_FINAL_actual.sh
    project_FINAL_actual_USE_THIS.sh

After a few weeks nobody knows which file is the real version.
You cannot see what changed between files.
You cannot go back to a specific working state.
If two people work on the same file, one person's changes 
get overwritten by the other.

----------------------------------------------------------------
THE SOLUTION
----------------------------------------------------------------

Version control is a system that tracks every change made to 
every file over time.

Every change is recorded with:
    - What changed (the actual lines added or removed)
    - Who made the change (name and email)
    - When it happened (date and time)
    - Why it was done (your commit message)

----------------------------------------------------------------
REAL LIFE ANALOGY
----------------------------------------------------------------

Think of it like a checkpoint system in a video game.

Every time you reach a safe point you save the game.
If you die, you go back to the last checkpoint.
You never lose all your progress and start from the beginning.

In the same way, version control lets you go back to any 
previous state of your project instantly.

----------------------------------------------------------------
THREE THINGS VERSION CONTROL SOLVES
----------------------------------------------------------------

1. GOING BACK IN TIME
   You can return to any previous version of any file at 
   any point in history.

2. TRACKING CHANGES
   You always know what changed, who changed it, and why.

3. WORKING IN TEAMS
   Multiple people can work on the same project at the same 
   time without overwriting each other's work.

----------------------------------------------------------------
KEY TAKEAWAY
----------------------------------------------------------------

Version control is not just for large teams or complex 
projects. Even a single developer working alone benefits 
from it because it gives you a complete safety net for 
every change you ever make.
